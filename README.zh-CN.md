## Paperclip 是什么？

# 零人工公司的开源编排平台

**如果 OpenClaw 是一名_员工_，那 Paperclip 就是一家_公司_**

Paperclip 是一个 Node.js 服务端 + React UI 平台，用于编排 AI Agent 团队来运营业务。接入你自己的 Agent，分配目标，在一个仪表板上追踪工作进展和成本。

它看起来像任务管理器——但底层拥有组织架构、预算管理、治理审批、目标对齐和 Agent 协调。

**管理业务目标，而不是 Pull Request。**

|        | 步骤     | 示例                                                               |
| ------ | -------- | ------------------------------------------------------------------ |
| **01** | 定义目标 | _"打造排名第一的 AI 笔记应用，实现百万月活。"_                     |
| **02** | 组建团队 | CEO、CTO、工程师、设计师、市场——任何机器人，任何供应商。           |
| **03** | 审批运行 | 审核策略、设定预算、启动运行，从仪表板监控。                       |

<br/>

> **即将推出：Clipmart** — 一键下载并运行完整的公司。浏览预构建的公司模板——包含完整的组织结构、Agent 配置和技能——并在几秒内导入到你的 Paperclip 实例中。

## Paperclip 适合你吗？

- ✅ 你想构建**自主运行的 AI 公司**
- ✅ 你需要**协调多个不同的 Agent**（OpenClaw、Codex、Claude、Cursor）朝着共同目标工作
- ✅ 你同时开着 **20 个 Claude Code 终端**，已经搞不清谁在做什么
- ✅ 你想让 Agent **7×24 小时自主运行**，但仍能在需要时审计工作和介入
- ✅ 你想**监控成本**并强制执行预算
- ✅ 你希望管理 Agent 的流程**像使用任务管理器一样简单**
- ✅ 你想**通过手机**管理你的自主业务

## 为什么 Paperclip 与众不同

Paperclip 正确处理了编排中最困难的细节。

|                                   |                                                                                                   |
| --------------------------------- | ------------------------------------------------------------------------------------------------- |
| **原子化执行。**                   | 任务签出和预算执行是原子操作，杜绝重复工作和成本失控。                                              |
| **持久化 Agent 状态。**            | Agent 在心跳之间恢复相同的任务上下文，而不是从零开始。                                              |
| **运行时技能注入。**               | Agent 可以在运行时学习 Paperclip 工作流和项目上下文，无需重新训练。                                  |
| **可回滚的治理。**                 | 审批关卡被强制执行，配置变更有版本控制，错误的变更可以安全回滚。                                     |
| **目标感知执行。**                 | 任务携带完整的目标谱系，Agent 始终能看到"为什么"，而不仅仅是一个标题。                               |
| **可移植的公司模板。**             | 导出/导入组织、Agent 和技能，支持密钥清除和冲突处理。                                               |
| **真正的多公司隔离。**             | 每个实体都按公司隔离，一次部署可以运行多家公司，拥有独立的数据和审计追踪。                            |

### 核心系统

<table>
<tr>
<td width="50%">

**身份与访问控制** — 两种部署模式（可信本地或认证模式），董事会用户、Agent API 密钥、短期运行 JWT、公司成员资格、邀请流程和 OpenClaw 入职。每个变更请求都可追溯到操作者。

</td>
<td width="50%">

**组织架构与 Agent** — Agent 拥有角色、头衔、汇报线、权限和预算。适配器示例包括：Claude Code、Codex、CLI Agent（如 Cursor/Gemini/Bash）、HTTP/Webhook 机器人（如 OpenClaw）以及外部适配器插件。只要能接收心跳，就是你的员工。

</td>
</tr>
<tr>
<td>

**工作与任务系统** — Issue 关联公司/项目/目标/父级链接，原子签出配合执行锁，一级阻塞依赖，评论、文档、附件、工作产品、标签和收件箱状态。杜绝重复工作，杜绝上下文丢失。

</td>
<td>

**心跳执行引擎** — 基于数据库的唤醒队列（含合并）、预算检查、工作空间解析、密钥注入、技能加载和适配器调用。运行产生结构化日志、成本事件、会话状态和审计追踪。恢复机制自动处理孤立运行。

</td>
</tr>
<tr>
<td>

**工作空间与运行时** — 项目工作空间、隔离的执行工作空间（git worktree、operator 分支）和运行时服务（开发服务器、预览 URL）。Agent 每次都在正确的目录和正确的上下文中工作。

</td>
<td>

**治理与审批** — 董事会审批工作流、带审核/审批阶段的执行策略、决策追踪、预算硬性停止、Agent 暂停/恢复/终止，以及完整的审计日志。你是董事会——没有你的签字，什么都不发布。

</td>
</tr>
<tr>
<td>

**预算与成本控制** — 按公司、Agent、项目、目标、Issue、供应商和模型追踪 Token 和成本。范围化预算策略，支持警告阈值和硬性停止。超支自动暂停 Agent 并取消排队工作。

</td>
<td>

**例程与定时调度** — 支持 cron、webhook 和 API 触发的定期任务。并发和追补策略。每次例程执行创建追踪工单并唤醒指定 Agent——无需手动触发。

</td>
</tr>
<tr>
<td>

**插件系统** — 实例级插件系统，支持进程外 Worker、基于能力的宿主服务、任务调度、工具暴露和 UI 贡献。无需 fork 即可扩展 Paperclip。

</td>
<td>

**密钥与存储** — 实例和公司密钥、加密本地存储、供应商支持的对象存储、附件和工作产品。敏感值不会出现在提示中，除非作用域运行明确需要。

</td>
</tr>
<tr>
<td>

**活动与事件** — 变更操作、心跳状态变更、成本事件、审批、评论和工作产品作为持久化活动记录，便于运营者审计发生了什么以及为什么。

</td>
<td>

**公司可移植性** — 导出和导入完整的组织——Agent、技能、项目、例程和 Issue——支持密钥清除和冲突处理。一次部署，多家公司，完全数据隔离。

</td>

## Paperclip 不是什么

|                               |                                                                                                     |
| ----------------------------- | --------------------------------------------------------------------------------------------------- |
| **不是聊天机器人。**           | Agent 有工作，而不是聊天窗口。                                                                       |
| **不是 Agent 框架。**          | 我们不教你如何构建 Agent。我们教你如何运营一家由 Agent 组成的公司。                                    |
| **不是工作流构建器。**         | 没有拖拽式流水线。Paperclip 建模的是公司——有组织架构、目标、预算和治理。                                |
| **不是提示词管理器。**         | Agent 自带提示词、模型和运行时。Paperclip 管理的是它们工作所在的组织。                                  |
| **不是单 Agent 工具。**        | 这是给团队用的。如果你只有一个 Agent，你可能不需要 Paperclip。如果你有二十个——你绝对需要。             |
| **不是代码审查工具。**         | Paperclip 编排的是工作，不是 Pull Request。请使用你自己的审查流程。                                    |

## 快速开始

开源。自托管。无需 Paperclip 账户。

```bash
npx paperclipai onboard --yes
```

快速入门默认使用可信本地回环模式，确保最快首次运行体验。如需以认证/私有模式启动，请明确选择绑定预设：

```bash
npx paperclipai onboard --yes --bind lan
# 或：
npx paperclipai onboard --yes --bind tailnet
```

如果已配置过 Paperclip，重新运行 `onboard` 会保留现有配置。使用 `paperclipai configure` 编辑设置。

手动安装：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

这将在 `http://localhost:3100` 启动 API 服务器。嵌入式 PostgreSQL 数据库会自动创建——无需额外设置。

> **系统要求：** Node.js 20+，pnpm 9.15+

## 常见问题

**典型部署是什么样的？**
本地运行时，单个 Node.js 进程管理嵌入式 PostgreSQL 和本地文件存储。生产环境中，指向你自己的 PostgreSQL，按你喜欢的方式部署。配置项目、Agent 和目标——Agent 会处理其余的事情。

如果你是独立开发者，可以使用 Tailscale 在外出时访问 Paperclip。之后需要时可以部署到 Vercel 等平台。

**可以运行多家公司吗？**
可以。单次部署可以运行不限数量的公司，且完全数据隔离。

**Paperclip 和 OpenClaw、Claude Code 等 Agent 有什么不同？**
Paperclip _使用_这些 Agent。它将它们编排成一家公司——拥有组织架构、预算、目标、治理和问责机制。

**为什么要用 Paperclip 而不是让 OpenClaw 对接 Asana 或 Trello？**
Agent 编排在协调谁签出了什么工作、维护会话、监控成本、建立治理等方面有很多细节——Paperclip 替你搞定这些。

（自带工单系统集成在路线图中）

**Agent 是持续运行吗？**
默认情况下，Agent 按计划心跳和基于事件的触发器运行（任务分配、@提及）。你也可以接入像 OpenClaw 这样的持续运行 Agent。你负责带来 Agent，Paperclip 负责协调。

## 开发

```bash
pnpm dev              # 完整开发（API + UI，监听模式）
pnpm dev:once         # 完整开发（无文件监听）
pnpm dev:server       # 仅服务端
pnpm build            # 构建所有
pnpm typecheck        # 类型检查
pnpm test             # 快速默认测试（仅 Vitest）
pnpm test:watch       # Vitest 监听模式
pnpm test:e2e         # Playwright 浏览器测试
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 应用迁移
```

`pnpm test` 不会运行 Playwright。浏览器测试套件独立运行，通常只在开发相关流程或在 CI 中运行。

详见 [doc/DEVELOPING.md](doc/DEVELOPING.md) 获取完整开发指南。

## 遥测

Paperclip 收集匿名使用遥测数据，帮助我们了解产品使用情况并持续改进。不会收集任何个人信息、Issue 内容、提示词、文件路径或密钥。私有仓库引用在发送前会使用每次安装唯一的盐值进行哈希处理。

遥测**默认开启**，可通过以下任一方式禁用：

| 方式         | 方法                                                         |
| ------------ | ------------------------------------------------------------ |
| 环境变量     | `PAPERCLIP_TELEMETRY_DISABLED=1`                             |
| 标准约定     | `DO_NOT_TRACK=1`                                             |
| CI 环境      | 当 `CI=true` 时自动禁用                                      |
| 配置文件     | 在 Paperclip 配置中设置 `telemetry.enabled: false`           |
